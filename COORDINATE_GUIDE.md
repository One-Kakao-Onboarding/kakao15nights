# UX-Ray 좌표 판단 가이드

## 📐 좌표 시스템 개요

UX-Ray는 **정규화된 좌표 시스템(0~1)**을 사용합니다.

### 좌표 기준
```
이미지 좌표계:
┌─────────────────────────┐
│ (0,0)          (1,0)    │  ← 상단
│                         │
│       요소               │
│    (x, y, w, h)         │
│                         │
│ (0,1)          (1,1)    │  ← 하단
└─────────────────────────┘
  ↑                    ↑
 좌측                 우측
```

### 좌표 값 의미
- **x**: 0 (왼쪽 끝) ~ 1 (오른쪽 끝)
- **y**: 0 (상단 끝) ~ 1 (하단 끝)
- **width**: 0 (없음) ~ 1 (전체 너비)
- **height**: 0 (없음) ~ 1 (전체 높이)

## 🎯 좌표 판단 팁

### 1. X 좌표 (가로 위치)
```
X 좌표 판단 방법:
┌─────────────────────────┐
│ 0.0   0.25   0.5   0.75  1.0
│  │      │     │      │    │
│  왼쪽   1/4   중앙   3/4  오른쪽
```

**예시:**
- 화면 왼쪽 끝 버튼: `x: 0.05`
- 화면 중앙 로고: `x: 0.45` (중앙이 0.5이므로 로고 시작점)
- 화면 오른쪽 버튼: `x: 0.85`

### 2. Y 좌표 (세로 위치)
```
Y 좌표 판단 방법:
┌──┐
│0.0│ ← 상단 헤더
├──┤
│0.2│ ← 상단 1/5 지점
├──┤
│0.5│ ← 중앙
├──┤
│0.8│ ← 하단 근처
├──┤
│1.0│ ← 최하단
└──┘
```

**예시:**
- 상단 네비게이션: `y: 0.0 ~ 0.1`
- 메인 콘텐츠: `y: 0.15 ~ 0.85`
- 하단 버튼: `y: 0.9 ~ 0.95`

### 3. Width & Height (크기)
```
크기 판단 방법:
- width: 0.1 = 전체 너비의 10%
- width: 0.5 = 전체 너비의 50%
- height: 0.05 = 전체 높이의 5%
```

**버튼 크기 예시:**
- 작은 아이콘 버튼: `width: 0.08, height: 0.04`
- 중간 버튼: `width: 0.2, height: 0.06`
- 큰 CTA 버튼: `width: 0.4, height: 0.08`

## 📱 모바일 vs 웹 판단 가이드

### 모바일 UI (세로형)
```
일반적인 모바일 레이아웃:
┌─────────────┐
│   Header    │ y: 0.0 ~ 0.1
├─────────────┤
│             │
│   Content   │ y: 0.1 ~ 0.85
│             │
├─────────────┤
│   Footer    │ y: 0.85 ~ 1.0
└─────────────┘
```

### 웹 UI (가로형)
```
일반적인 웹 레이아웃:
┌──────┬──────────────┐
│ Nav  │   Header     │ y: 0.0 ~ 0.15
│      ├──────────────┤
│ x:0  │   Content    │ y: 0.15 ~ 0.9
│ ~0.2 │              │
│      ├──────────────┤
│      │   Footer     │ y: 0.9 ~ 1.0
└──────┴──────────────┘
```

## 🎨 페르소나별 좌표 판단 포인트

### 👵 할머니 페르소나 (노안/저시력)
**주목할 영역:**
```
작은 텍스트 위치:
- 상단 헤더 (y: 0.0 ~ 0.1)
- 푸터 정보 (y: 0.9 ~ 1.0)
- 서브 메뉴 (x: 0.8 ~ 1.0, y: 0.0 ~ 0.15)
```

**프롬프트 예시:**
```javascript
"이 UI에서 작은 텍스트를 찾아주세요.
좌표는 정규화된 값(0~1)으로 제공하세요.

예시:
- 상단 '도움말' 링크가 화면 우상단 (x: 0.85, y: 0.02)에 있고
  크기가 작다면 (width: 0.1, height: 0.03)

응답:
{
  'coordinates': { 'x': 0.85, 'y': 0.02, 'width': 0.1, 'height': 0.03 }
}"
```

### 🧠 ADHD 페르소나 (주의력 산만)
**주목할 영역:**
```
산만한 요소들:
- 여러 개의 버튼들
- 팝업이나 배너
- 애니메이션 요소
```

**좌표 판단 팁:**
```
복수 요소 표시:
[
  { "x": 0.1, "y": 0.3, "width": 0.15, "height": 0.05 },  // 버튼 1
  { "x": 0.3, "y": 0.3, "width": 0.15, "height": 0.05 },  // 버튼 2
  { "x": 0.5, "y": 0.3, "width": 0.15, "height": 0.05 }   // 버튼 3
]
```

### ✋ 한손사용자 페르소나
**주목할 영역:**
```
엄지 도달 존 (모바일 기준):
┌─────────────┐
│ 🔴 Pain     │ y: 0.0 ~ 0.3  (빨강 - 닿기 어려움)
├─────────────┤
│ 🟡 Stretch  │ y: 0.3 ~ 0.6  (노랑 - 불편함)
├─────────────┤
│ 🟢 Comfort  │ y: 0.6 ~ 1.0  (초록 - 편함)
└─────────────┘
```

**프롬프트 팁:**
```
"상단의 뒤로가기 버튼 (y: 0.05) 은 한 손으로 닿기 어렵습니다.
하단의 확인 버튼 (y: 0.85) 은 엄지로 쉽게 누를 수 있습니다."
```

### 🌍 외국인 페르소나
**주목할 영역:**

- 배너 이미지 내 텍스트

## 💡 정확한 좌표 계산 방법

### 방법 1: 시각적 추정
```
1. 이미지를 9등분으로 나눔
┌───┬───┬───┐
│0.0│0.3│0.6│
├───┼───┼───┤
│0.0│0.3│0.6│
├───┼───┼───┤
│0.0│0.3│0.6│
└───┴───┴───┘

2. 요소 위치 확인
   - 왼쪽 1/3: x ≈ 0.15
   - 중앙 1/3: x ≈ 0.5
   - 오른쪽 1/3: x ≈ 0.85
```

### 방법 2: 상대적 위치 계산
```
예시: 버튼이 화면 너비의 20% 지점에 있고,
      버튼 너비가 화면의 30%라면

x = 0.2          (시작점)
width = 0.3      (너비)
y = 0.7          (상단으로부터 70% 지점)
height = 0.08    (높이는 화면의 8%)
```

### 방법 3: 그리드 기준
```
12 컬럼 그리드 시스템 변환:
- 1 컬럼 = width: 0.083 (1/12)
- 2 컬럼 = width: 0.167 (2/12)
- 3 컬럼 = width: 0.25  (3/12)
- 6 컬럼 = width: 0.5   (6/12)
```

## 🚨 흔한 실수와 해결법

### ❌ 실수 1: 픽셀 좌표 사용
```javascript
// 잘못된 예
{ "x": 150, "y": 200, "width": 100, "height": 50 }

// 올바른 예 (이미지가 1000x800이라면)
{ "x": 0.15, "y": 0.25, "width": 0.1, "height": 0.0625 }
```

### ❌ 실수 2: 1보다 큰 값
```javascript
// 잘못된 예
{ "x": 1.5, "y": 0.8 }  // x가 화면 밖

// 올바른 예
{ "x": 0.95, "y": 0.8 }  // 화면 오른쪽 끝 근처
```

### ❌ 실수 3: 음수 값
```javascript
// 잘못된 예
{ "x": -0.1, "y": 0.5 }  // x가 음수

// 올바른 예
{ "x": 0.0, "y": 0.5 }   // 왼쪽 끝
```

### ❌ 실수 4: 너무 정확한 좌표
```javascript
// 불필요하게 정확함
{ "x": 0.4738291, "y": 0.2847362 }

// 적절한 정확도 (소수점 2자리)
{ "x": 0.47, "y": 0.28 }
```

- [ ] width와 height가 0보다 큰가?

  시작: (${testCoord.x * imageWidth}, ${testCoord.y * imageHeight})

## 📚 실제 프롬프트 예시

### 완전한 프롬프트 구조
```
당신은 UX 분석가입니다. 이 UI 화면을 분석하여 문제점을 찾아주세요.

[페르소나 특성 설명]

[찾아야 할 문제점 목록]

**좌표 제공 가이드:**
- 좌표는 0~1 사이의 정규화된 값으로 제공
- x: 이미지 너비 기준 0(왼쪽)~1(오른쪽)
- y: 이미지 높이 기준 0(위)~1(아래)
- width/height: 전체 크기 대비 비율

**좌표 예시:**
화면 중앙에 있는 버튼:
{ "x": 0.4, "y": 0.5, "width": 0.2, "height": 0.06 }

화면 우상단 닫기 버튼:
{ "x": 0.9, "y": 0.05, "width": 0.08, "height": 0.05 }

응답 형식:
{
  "score": 3.5,
  "issues": [
    {
      "title": "문제 제목",
      "description": "구체적인 설명",
      "coordinates": { "x": 0.1, "y": 0.2, "width": 0.3, "height": 0.1 },
      "severity": "high"
    }
  ],
  "summary": "전반적인 평가"
}
```

### 참고할 만한 좌표 범위
```
모바일 UI 기준:
- 상단 상태바/헤더: y: 0.0 ~ 0.12
- 메인 콘텐츠: y: 0.12 ~ 0.85
- 하단 네비게이션: y: 0.85 ~ 1.0
- 플로팅 버튼: x: 0.8 ~ 0.95, y: 0.8 ~ 0.95

웹 UI 기준:
- 헤더: y: 0.0 ~ 0.1
- 사이드바: x: 0.0 ~ 0.2 또는 x: 0.8 ~ 1.0
- 메인 콘텐츠: x: 0.2 ~ 0.8 (사이드바 있을 때)
- 푸터: y: 0.9 ~ 1.0
```
